# 메타데이터 담당 — 작업 순서

`scripts/main.py`·`pure_metadata_export.py` 기준으로, **파일 이름**과 **할 일 순서**를 한곳에 정리한 문서다.

---

## 1. 산출물 파일 이름 (현재 코드 기준)

| 파일 | 역할 |
|------|------|
| **`metadata.jsonl`** | **본산출물 (DB·품질)**. 한 줄에 `source_pdf`, `title`, `bc_id`, `sc_keyword`, `tl_summary` 등 **순수 메타데이터**. |
| **`finetune_dataset.jsonl`** | **파인튜닝용**. `instruction` / `input` / `output` 형식. `main.py`에서 `WRITE_FINETUNE = True`일 때만 생성. |
| **`dataset.jsonl`** | **예전 이름·레거시**. 역할은 **`finetune_dataset.jsonl`과 같은 줄**(학습용 instruction/input/output)로 보면 된다. 새 작업에서는 **`finetune_dataset.jsonl`**을 쓰는 것을 권장. |
| **`metadata_db.jsonl`** (선택) | `pure_metadata_export.py` 기본 출력. 레거시 `dataset.jsonl` 등에서 DB 스키마만 뽑을 때 사용. |

**요약:** 품질·DB는 **`metadata.jsonl`**이 기준. 학습 데이터는 **`finetune_dataset.jsonl`**.

---

## 2. 내가 할 일 — 순서

### 1단계: 데이터 만들기

1. `pdfdata/`에 PDF 넣기.
2. Ollama 기동·모델 확인 (`main.py`의 `OLLAMA_URL`, `OLLAMA_MODEL`).
3. `metadata` 폴더에서 실행:
   ```bash
   python scripts/main.py
   ```
4. 결과 확인:
   - `metadata.jsonl` (필수)
   - `finetune_dataset.jsonl` (학습 예정이면, `WRITE_FINETUNE = True` 유지)

### 2단계: 품질 기준 맞추기 (1단계 안에서 계속 다듬기)

- **`bc_id`**: 1~10만 허용되도록 모델+`normalize_bc_id`·`assign_bc_id` 동작 확인.
- **`sc_keyword`**: 형식(쉼표 구분, 최대 2개 등) 팀 기준과 일치하는지.
- **`tl_summary`**: 세 문장·마침표 등 **팀 정의**와 맞는지 (한국어는 마침표 없이 끝나는 경우도 있어서 기준을 문서로 정해 두면 좋음).

### 3단계: 검증

1. **샘플 100~200건**을 `metadata.jsonl` 기준으로 눈으로 검수.
2. 틀린 행은 **수정본**을 따로 두거나, 같은 형식으로 고쳐서 다시 합친 뒤 **버전 관리**(예: `metadata.reviewed.jsonl`).
3. **건수·누락 필드** 통계 (빈 `tl_summary`, 이상한 `bc_id` 등).

### 4단계: 파인튜닝·어댑터

1. 학습 포맷은 **`finetune_dataset.jsonl`** (또는 검수 반영한 동일 스키마 파일).
2. 베이스 모델·LoRA 하이퍼파라미터는 팀 표준 문서 따름.
3. 학습 후 **새 PDF**로만 추론 테스트 (학습에 쓴 PDF와 겹치지 않게).
4. 노트북은 분리해서 사용:
   - Colab: `finetune/colab_train.ipynb`
   - Kaggle: `finetune/kaggle_train.ipynb`
5. 학습 완료 즉시 허브 업로드가 필요하면 `ENABLE_UPLOAD=True` + `HF_REPO_ID` 설정 후, `HF_TOKEN`(Secrets)으로 자동 업로드 수행.

**Colab만 쓸 때 체크 (상세는 `finetune/README-colab-kaggle.md`):**

- 로컬에서 `WRITE_FINETUNE = True`로 **`finetune_dataset.jsonl`**까지 만든 뒤 업로드한다. Ollama·PDF 처리는 Colab에서 하지 않는다.
- Colab: GPU 런타임, `pip install`, 경로(`DATA_PATH` / `OUTPUT_DIR` 또는 `--data` / `--out`) 수정, 게이트 모델이면 **`HF_TOKEN`**.

### 5단계: 적용 (메타 담당 범위 밖과의 경계)

- 추론 서버(Ollama·vLLM 등)에 어댑터/모델 올리기 → **인프라·백엔드**와 협의.
- DB·Chroma 메타 키(`file_id` 등)와 **`metadata.jsonl` 필드 매핑** → **스키마 표**로 팀과 합의.
- 인제스트·권한 필터는 **Worker/메인 레포** 체크리스트 참고.

---

## 3. 보조 스크립트

| 용도 | 명령 예 |
|------|---------|
| 레거시/학습용 JSONL → DB용 한 줄 형식만 추출 | `python scripts/pure_metadata_export.py --in dataset.jsonl --out metadata_db.jsonl` |
| 입력 생략 시 `metadata.jsonl` 우선, 없으면 `dataset.jsonl` | `python scripts/pure_metadata_export.py` |

---

## 4. 한 줄 요약

**만들기 → `metadata.jsonl`로 품질 확보·검수 → (필요 시) `finetune_dataset.jsonl`로 학습 → 새 PDF로 평가 → 팀 스키마에 맞춰 DB·워커에 연결.**

이 문서는 `scripts/main.py` 동작과 맞출 때 업데이트하면 된다.
